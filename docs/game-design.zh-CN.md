# 游戏逻辑设计

## 概述

当前项目包含两种可玩模式：

1. `三段闯关` 单人模式
2. `房间竞速` 多人模式

本文档描述的是当前代码中已经实现并生效的游戏逻辑。

## 三段闯关

### 目标

玩家先选择目标距离，再把这段距离拆成 3 个前进阶段。
系统会生成木头人的 3 个回头检查点。
每个阶段都会用玩家的累计路程与对应检查点比较。
3 个阶段全部通过则获胜。
任意一个阶段失败则整局立即结束。

### 输入规则

玩家需要选择：

1. `target_distance`
   范围：`3` 到 `30`
2. `segments`
   必须是 3 个正整数，且三者之和等于 `target_distance`

示例：

- `3 -> [1, 1, 1]`
- `4 -> [1, 1, 2]`、`[1, 2, 1]`、`[2, 1, 1]`
- `6 -> [1, 2, 3]`、`[2, 2, 2]` 等

### 检查点生成规则

每局会生成 3 个检查点数值，记作 `turn_values`。

规则如下：

1. 3 个检查点都在 `1 .. target_distance` 范围内
2. 3 个检查点互不重复
3. 3 个检查点按从小到大排序
4. 第 3 个检查点固定等于 `target_distance`

示例：

- `target_distance = 3` -> `[1, 2, 3]`
- `target_distance = 6` -> `[1, 4, 6]`、`[2, 3, 6]`
- `target_distance = 20` -> `[4, 11, 20]`

这 3 个数表示前进路线上的 3 个检查点。

### 阶段距离与累计路程

玩家选择的 3 个数字表示每个阶段各自前进的距离。
真正用于判定的是累计路程，不是单段距离。

示例：

- `segments = [1, 4, 1]`

累计路程为：

- 第 1 阶段：`1`
- 第 2 阶段：`1 + 4 = 5`
- 第 3 阶段：`1 + 4 + 1 = 6`

### 胜负判定

设：

- 阶段距离为 `[s1, s2, s3]`
- 累计路程为 `[d1, d2, d3]`
- 检查点为 `[t1, t2, t3]`

其中：

- `d1 = s1`
- `d2 = s1 + s2`
- `d3 = s1 + s2 + s3`

通过条件为：

- `d1 <= t1`
- `d2 <= t2`
- `d3 <= t3`

当前比较规则使用 `<=`。
也就是说，累计路程刚好到达检查点时也算通过。

### 中途失败立即结束

只要任意一个阶段失败，整局就会立刻结束。
系统不会继续处理后续阶段。

示例：

- 第 1 阶段累计路程：`1`
- 第 2 阶段累计路程：`5`
- 第 2 个检查点：`4`
- 判定结果：`5 > 4`

此时玩家在第 2 阶段失败，第 3 阶段不再计算。

### 获胜条件

玩家只有在 3 个阶段全部通过时才会获胜。

因为最后一个检查点固定等于 `target_distance`，所以最后一段成功通过就表示玩家在没有被抓到的情况下到达了终点。

### 奖励倍率

奖励倍率只与 `target_distance` 有关。
与玩家选择了哪一种三段拆分无关。

当前实现使用线性倍率表：

- `3 -> 1.0x`
- `4 -> 1.2x`
- `5 -> 1.4x`
- ...
- `30 -> 6.4x`

### 单人模式接口

当前接口如下：

- `GET /api/game/config`
- `GET /api/game/combinations/{distance}`
- `POST /api/game/start`
- `GET /api/game/{game_id}`

其中 `POST /api/game/start` 会立即完成：

- 输入校验
- 检查点生成
- 对局结算
- 结果内存存储

### 单人模式前端表现

单人模式是首页主内容。

当前前端行为包括：

- 使用滑块选择目标距离
- 选择一个合法的三段组合
- 在游戏画面中直接开始本局
- 在赛道上预览累计路程检查点
- 按阶段播放角色前进动作
- 失败时立即停止播放
- 使用可折叠的“最近一局”面板查看详细结果

角色头顶提示和结果面板的比较展示规则为：

- 通过时显示 `<=`
- 失败时显示 `>`

## 房间竞速

### 目标

玩家创建或加入房间，必要时添加机器人，并在红灯停、绿灯行的规则下完成竞速。

### 房间流程

1. 房主创建房间
2. 其他真人玩家通过房间号加入
3. 创建房间时可加入机器人
4. 房主开始比赛
5. 玩家在绿灯阶段前进
6. 红灯阶段移动会立即失败
7. 所有位置与结果通过实时同步更新

### 多人模式接口

当前接口如下：

- `GET /api/rooms`
- `POST /api/rooms/create`
- `POST /api/rooms/{room_code}/join`
- `GET /api/rooms/{room_code}`
- `POST /api/rooms/{room_code}/start`
- `POST /api/rooms/{room_code}/players/{player_id}/move`
- `GET /ws/rooms/{room_code}`

### 多人模式前端入口

当前多人入口通过按钮提供：

- `创建房间`
- `可加入房间`

两者都通过弹窗打开，不再长期占据首页空间。

## 当前逻辑的来源

当前项目中：

- 后端 `src/backend/main.py` 是玩法规则的事实来源
- 前端 `src/frontend/src/App.vue` 负责把这些规则可视化表现出来

当前核心规则总结如下：

1. 单人模式由 3 个阶段距离构成
2. 判定使用累计路程
3. 检查点互不重复且递增
4. 最后一个检查点固定等于终点距离
5. 比较规则为 `<=`
6. 任意阶段失败后整局立即结束
7. 前端表现已与后端结算逻辑保持一致
