# 当前系统胜率分析报告

## 1. 目的

本文基于项目在 `2026-05-15` 时的**当前实际实现**，对单人玩法的玩家胜率进行统计分析，给出：

- 当前规则说明
- 整体胜率
- 分目标距离平均胜率
- 强组合与弱组合特征
- 当前规则的平衡性判断

---

## 2. 当前系统规则

本报告严格按照当前后端真实实现进行计算。

### 2.1 当前规则摘要

1. 目标距离范围为 `6-30`
2. 玩家选择 3 段组合 `a, b, c`
3. 每段允许为 `0` 或正整数
4. 要求三段和满足：
   - `a + b + c = N`
5. 木头人每局生成 3 个回头数值 `t1, t2, t3`
6. 回头数值规则：
   - 从 `1 ~ N-1` 中生成
   - 不包含起点 `0`
   - 不包含终点 `N`
   - 三个值严格递增
   - 三个值互不重复

### 2.2 判定规则

当前系统的 3 次比较方式不是完全一致的。

#### 第 1 段

- 比较第 1 段累计路程
- 判定为：
  - `a <= t1`

#### 第 2 段

- 比较前两段累计路程
- 判定为：
  - `a + b <= t2`

#### 第 3 段

- 不再比较到终点的累计路程
- 改为比较第三段步长
- 判定为：
  - `c <= t3`

### 2.3 失败规则

- 任意一段失败，整局立即失败
- 后续阶段不再继续判定

---

## 3. 统计口径

本报告采用两种统计口径。

### 3.1 口径 A：全量组合池等概率

- 将 `6-30` 的所有合法三段组合放在一起
- 默认每个组合出现概率相同

这个口径更适合观察“当前整套规则池”的总体胜率。

### 3.2 口径 B：先等概率选距离，再等概率选组合

- 先在 `6-30` 之间等概率选择目标距离
- 再在该距离下等概率选择一个合法三段组合

这个口径更接近“每个距离档位权重相同”的产品观察视角。

---

## 4. 总体结论

### 4.1 整体胜率

- 口径 A：`17.88%`
- 口径 B：`18.56%`

这说明：

- 如果把所有合法组合全部平铺看，玩家平均胜率约为 `17.88%`
- 如果按距离档位均匀看，玩家平均胜率约为 `18.56%`

结论非常明确：

- 当前系统整体已经属于**偏难规则**

## 6. 强组合与弱组合

### 6.1 当前最强组合特征

当前规则下，仍然存在明显偏强的组合。

一些代表性示例如下：

- `6` 米：`[0, 2, 4]`，胜率 `90%`
- `7` 米：`[0, 2, 5]` 或 `[0, 3, 4]`，胜率 `80%`
- `8` 米：`[0, 3, 5]` 或 `[1, 2, 5]`，胜率 `80%`
- `10` 米：`[0, 4, 6]`，胜率约 `73.81%`
- `20` 米：`[0, 7, 13]`，胜率约 `66.46%`
- `30` 米：`[0, 11, 19]`，胜率约 `64.12%`

这些组合的共同特征是：

1. 第 1 段通常很小，甚至为 `0`
2. 第 2 段累计路程控制在较容易通过的区间
3. 第 3 段虽然较大，但仍能压在最后一个回头值的高概率区间内

### 6.2 当前最弱组合特征

当前规则下，也存在大量**天然必输**的组合。

典型示例：

- `[0, 0, 6]`
- `[0, 0, 10]`
- `[0, 0, 20]`
- `[0, 0, 30]`

这类组合的胜率为：

- `0%`

原因非常直接：

1. 前两段都是 `0`
2. 第三段步长等于整个目标距离
3. 但最后一个回头值只会落在 `1 ~ N-1`
4. 因此最后一段一定满足：
   - `N > t3`
5. 第三段必败

也就是说：

- `[0, 0, N]` 在当前系统中属于结构性必输组合

## 7. 当前规则的平衡性判断

### 7.1 已经修掉了旧版的终点必胜问题

旧版规则的问题是：

- 最后一个回头点固定等于终点
- 容易出现结构性必胜组合

当前系统已经做了两项关键修正：

1. 回头值不再包含终点
2. 第三段改成比较步长，而不是最终累计路程

因此：

- 旧版那种“终点天然通过”的漏洞已经被修掉

### 7.2 但规则仍然存在明显的组合极差

虽然旧漏洞已经修复，但当前系统仍然存在新的结构性问题：

- 一些组合非常强，甚至可达 `100%`
- 一些组合非常弱，甚至固定为 `0%`

这意味着：

- 组合之间的胜率差异仍然过大
- 玩家一旦理解规则，会很快放弃大量弱组合

### 7.3 整体难度偏高

按整体统计结果看：

- 当前总胜率只有 `17.88% ~ 18.56%`

这已经明显低于“中等挑战”的常见体验区间。

如果希望玩法更偏大众化，当前系统会显得：

- 挑战较强
- 容错较低
- 不熟悉规则的玩家容易连续失败

### 7.4 “允许两个 0” 会进一步拉大组合强弱差距

当前规则允许组合中出现 `2` 个 `0`，这会导致：

- 一部分组合把压力全部堆到第三段，变成必输
- 另一部分组合则能把前两段压得非常轻，从而显著提高前两段通过率

因此，“允许两个 `0`”本身就是导致极端组合大量出现的重要原因之一。

---

## 8. 结论摘要

当前系统的关键结论如下：

1. 当前整体胜率约为 `17.88% ~ 18.56%`
2. 目标距离越大，平均胜率越低
3. 高距离区间胜率稳定在 `17%-19%` 左右
4. 旧版“终点固定回头点”的漏洞已经被修掉
5. 但当前仍然存在明显偏强组合和结构性必输组合
6. 当前规则整体偏难，且组合之间强弱差距仍然较大

---

## 9. 建议

如果后续希望把玩法调到更健康的区间，建议优先考虑以下方向：

1. 限制最多只允许出现 `1` 个 `0`
2. 重新分布木头人 3 个回头值的生成区间
3. 降低第三段“天然承压”的程度
4. 重新校准距离倍率与真实胜率之间的匹配关系

如果目标是把整体胜率提升到更适中的区间，例如 `25%-35%`，那么最优先的调整方向通常是：

- 限制极端组合
- 调整回头值分布
