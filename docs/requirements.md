# 产品需求描述文档

## 1. 产品概述

Freeze123 是一个基于浏览器运行的 HTML5 小游戏，玩法来自经典的“一二三木头人”。玩家需要在允许移动的阶段向终点前进，并在禁止移动的阶段保持静止。如果玩家在禁止移动阶段继续移动，则游戏失败；如果玩家成功到达终点，则游戏胜利。

项目首版目标是实现一个轻量、可玩的单人版本，并使用 Vue 构建前端交互界面，使用 Python FastAPI 提供后端请求响应接口。

## 2. 产品目标

- 提供一个可以直接在浏览器中体验的“一二三木头人”小游戏。
- 完成从开始游戏、移动、判定失败、判定胜利到展示结果的完整闭环。
- 前端负责游戏画面、交互输入、状态展示和基础动画。
- 后端负责游戏会话创建、状态计算、结果返回和后续扩展入口。
- 项目结构清晰，方便后续扩展排行榜、难度选择、用户系统和更多关卡。

## 3. 目标用户

### 3.1 普通玩家

普通玩家希望打开网页即可开始游戏，不需要登录或复杂配置。玩家关注游戏是否简单易懂、反馈是否清晰、操作是否顺畅。

### 3.2 开发和学习用户

开发者或学习者希望通过该项目了解 Vue 前端与 FastAPI 后端如何配合，实现一个小型游戏项目的完整请求响应流程。

## 4. 核心玩法

游戏分为“绿灯”和“红灯”两个阶段：

- 绿灯阶段：玩家可以移动，目标是尽快接近终点。
- 红灯阶段：玩家必须停止移动，如果继续移动则失败。
- 胜利条件：玩家到达终点线。
- 失败条件：玩家在红灯阶段产生移动行为。
- 游戏结果：展示胜利或失败状态，以及本局用时、移动距离或分数。

## 5. 功能需求

### 5.1 游戏开始

- 用户进入页面后可以看到游戏标题、开始按钮和基础提示。
- 点击开始按钮后创建一局新游戏。
- 前端请求后端创建游戏会话，并获得初始游戏状态。

### 5.2 游戏进行

- 页面展示玩家、起点、终点和当前灯光状态。
- 玩家可以通过键盘或页面按钮控制角色移动。
- 游戏应周期性切换绿灯和红灯状态。
- 当前状态变化需要有明显视觉反馈，例如颜色、文字或提示音效占位。

### 5.3 移动判定

- 绿灯阶段移动时，玩家位置向终点推进。
- 红灯阶段移动时，判定游戏失败。
- 玩家到达或超过终点线时，判定游戏胜利。
- 胜利或失败后，禁止继续移动。

### 5.4 结果展示

- 游戏结束后展示本局结果。
- 结果至少包含胜利或失败状态。
- 可选展示本局用时、最终距离、移动次数或分数。
- 用户可以点击“重新开始”开始新一局。

### 5.5 后端接口

首版后端提供基础请求响应能力：

- 健康检查接口，用于确认服务正常运行。
- 创建游戏接口，用于初始化一局游戏。
- 提交动作接口，用于提交玩家移动行为并返回最新状态。
- 查询状态接口，用于获取当前游戏状态。

具体接口字段将在 `docs/api.md` 中定义。

## 6. 非功能需求

### 6.1 易用性

- 游戏入口应简单清晰。
- 玩家不需要阅读复杂说明即可理解基础操作。
- 游戏状态、胜负结果和重新开始入口应明显可见。

### 6.2 响应速度

- 前端操作应有即时反馈。
- 后端接口响应应保持轻量，首版不依赖复杂数据库。
- 单局游戏状态可以先保存在内存中。

### 6.3 兼容性

- 首版优先支持桌面浏览器。
- 页面应适配常见屏幕宽度。
- 后续可扩展移动端触控操作。

### 6.4 可维护性

- 前端游戏逻辑应拆分清晰，避免所有逻辑堆在单一组件中。
- 后端接口、状态模型和业务规则应保持明确。
- 文档应随着接口和玩法变更同步更新。

## 7. 首版范围

### 7.1 包含范围

- Vue 前端游戏页面。
- FastAPI 后端基础接口。
- 单人游戏流程。
- 红灯、绿灯状态切换。
- 玩家移动、胜利判定和失败判定。
- 游戏结果展示。
- 重新开始功能。

### 7.2 暂不包含

- 用户注册和登录。
- 数据库存储。
- 在线多人对战。
- 排行榜。
- 复杂角色皮肤或商城系统。
- 关卡编辑器。
- 完整音效和美术资源系统。

## 8. 数据和状态

单局游戏建议包含以下状态：

- `game_id`：游戏会话 ID。
- `status`：游戏状态，例如 `waiting`、`running`、`won`、`lost`。
- `light`：当前灯光状态，例如 `green` 或 `red`。
- `player_position`：玩家当前位置。
- `finish_position`：终点位置。
- `started_at`：游戏开始时间。
- `ended_at`：游戏结束时间。
- `move_count`：玩家移动次数。
- `message`：给前端展示的状态提示。

## 9. 验收标准

- 用户可以从页面点击开始并进入游戏。
- 游戏过程中可以看到红灯和绿灯状态变化。
- 玩家在绿灯阶段移动时，位置正常前进。
- 玩家在红灯阶段移动时，游戏立即失败并展示失败结果。
- 玩家到达终点后，游戏胜利并展示胜利结果。
- 游戏结束后不能继续改变本局状态。
- 用户可以重新开始一局新游戏。
- FastAPI 后端至少提供健康检查、开始游戏、提交动作和查询状态接口。
- 前端能够通过 API 与后端完成基本游戏状态同步。

## 10. 后续迭代方向

- 增加排行榜和历史记录。

- 增加移动端触控操作。
- 增加音效、动画和更完整的视觉表现。
